### 발견 사항

1. **[SHOULD] /sources 페이지 title = "Submit Tool"**
   - `/sources` 경로의 SEO title이 "Submit Tool"로 설정돼 있다. URL 슬러그(`/sources`)와 title 사이에 의미 불일치가 있어 Google이 페이지 목적을 다른 방식으로 해석할 수 있다.
   - description은 툴 제출 양식에 맞게 작성돼 있어, 페이지 역할 자체는 명확하나 title을 `"Submit a Tool — VibeHub"` 또는 `"Sources"` 형태로 조정하는 것이 권장된다.

2. **[SHOULD] brief cover image에 raw `<img>` 태그 사용 (brief/[slug]/page.tsx L.176)**
   - `next/image` 대신 `<img>` 태그 직접 사용 중. LCP(Largest Contentful Paint) 최적화 누락, 자동 WebP 변환·srcset 생성 불가.
   - eslint-disable 주석이 이미 달려 있어 의도적으로 우회한 것이지만, next/image로 전환하면 Core Web Vitals 개선 가능.

3. **[SHOULD] YouTube embed에 대한 `<link rel="preconnect">` 미설정**
   - brief 상세 페이지에서 `https://www.youtube.com/embed/...` iframe을 조건부 렌더링하나, root layout에 `preconnect` 힌트 없음.
   - `<link rel="preconnect" href="https://www.youtube.com">` 추가 시 첫 TCP 연결 지연 감소.

4. **[SHOULD] ES 언어 RSS feed 없음**
   - `/feed.xml`은 EN 전용이며 EN brief 링크(`/en/brief/...`)만 제공한다. ES 독자를 위한 feed가 없다. 즉시 인덱싱에 영향은 없지만, ES 배포 확대 시 고려 필요.

5. **[SHOULD] /brief, /radar 리스트 페이지 ItemList 스키마 미구현**
   - 리스트 페이지에 `ItemList` JSON-LD 스키마가 없다. Google의 리치 리절트(특히 carousel) 노출 기회를 일부 놓치고 있다.
   - 우선순위가 높지는 않으나, brief 수가 50+ 이상으로 증가 시 중요도 상승.

6. **[INFO] published discover_items = 0**
   - Radar(discover_items) 공개 발행 건수가 0이다. 사이트맵과 일치하나, Radar 인덱싱 효과는 없는 상태. 의도된 상황이면 무시.
