# 세션 회고 → 스킬/피드백 자동 추출

이 세션에서 수행한 작업 전체를 회고하여, 재사용 가능한 패턴을 스킬·피드백·자동화로 추출한다.

## 분석 대상

대화 전체를 스캔하여 아래 4가지 신호를 찾는다:

### 1. 성공 패턴 (→ 스킬 후보)
- 3단계 이상의 반복 가능한 작업 흐름이 성공한 경우
- 여러 도구를 조합한 해결법이 효과적이었던 경우
- 병렬 에이전트 전략이 잘 작동한 경우
- **신호**: "완료", "성공", "통과", 에이전트 결과가 긍정적

### 2. 실패 → 우회 패턴 (→ 피드백 또는 스킬 보강)
- 처음 시도가 실패하고 다른 접근으로 해결한 경우
- 도구 한계를 발견하고 workaround를 찾은 경우
- **신호**: "실패", "안 됨", 에러 메시지 → 이후 다른 접근 시도

### 3. 반복 수작업 (→ 자동화 후보)
- 같은 종류의 작업을 3회 이상 수동 반복한 경우
- 사람이 개입해야 했던 판단이 규칙화 가능한 경우
- **신호**: 유사한 도구 호출이 반복, 비슷한 SQL/fetch 패턴

### 4. 사용자 교정 (→ 피드백)
- `/learn`으로 이미 저장한 것은 제외
- 사용자가 방향을 바꾼 지점, 불만족 표현, 판단 위임

## 출력 형식

분석 결과를 아래 표로 정리한다:

```
| # | 유형 | 내용 요약 | 액션 |
|---|------|----------|------|
| 1 | 성공패턴 | og:image 추출 실패 → GitHub social preview fallback 체인 | 스킬 신규/보강 |
| 2 | 실패우회 | WebFetch로 JS-heavy 사이트 og:image 추출 불가 → curl HEAD 검증 | 피드백 저장 |
| 3 | 반복수작업 | 이미지 URL 98개 수동 검증 | 자동화 신규 |
| 4 | 사용자교정 | 관련 없는 placeholder 이미지 거부 | 피드백 저장 |
```

## 실행 프로세스

1. 대화 전체를 분석하여 위 4가지 신호를 추출한다
2. 표로 정리하여 사용자에게 보여준다
3. 사용자 승인을 받는다 ("전부 적용", "N번만", "스킵")
4. 승인된 항목에 대해:

### 스킬 신규
- `.claude/skills/{name}/SKILL.md` 생성
- `CLAUDE.md`에 등록

### 스킬 보강
- 기존 SKILL.md에 단계/팁 추가
- 변경 내용 한 줄 리포트

### 피드백 저장
- `~/.claude/projects/.../memory/feedback_{topic}.md` 생성/업데이트
- `MEMORY.md` 인덱스 갱신

### 자동화 신규
- `.claude/automations/{name}.md` 생성
- `CLAUDE.md`와 `automations/README.md`에 등록

5. 적용 결과를 한 줄씩 리포트한다

## 규칙

- 코드 경로, 특정 UUID 등 휘발성 정보는 저장하지 않음 — 행동 원칙과 전략만
- 기존 스킬/피드백과 중복되면 기존 파일을 업데이트
- 너무 프로젝트 특화된 패턴은 스킬 대신 피드백으로 저장
- 범용적인 패턴은 스킬로 승격
- 사용자 승인 없이 파일을 생성/수정하지 않음
